# Atividades executadas na aula de hoje

- Na aula anterior o sistema foi dividido em módulos para criação das histórias de usuário
- Wellington --> Módulo de relatórios
- Na data de hoje a aula foi usada para revisar as histórias de usuario e adicionar novas histórias
- Foi dado continuidade na criação de histórias de usuários

1. Histórias de usuário relatórios

Como bibliotecária  
quero gerar relatórios do sistema  
Para acompanhar empréstimos, multas e histórico da biblioteca  

1. Cenários

Dado que a bibliotecária acessa a área de relatórios  
Quando ela escolhe entre relatório de empréstimos, multas ou histórico  
Então o sistema mostra os dados conforme o tipo selecionado  

Dado que o relatório foi gerado  
Quando a bibliotecária precisar usar fora do sistema  
Então deve ser possível exportar em PDF ou Excel com os mesmos dados exibidos  

Dado que a bibliotecária acessa a área de relatórios  
Quando não seleciona nenhum tipo de relatório  
Então o sistema deve solicitar a escolha de um tipo antes de continuar   

---

2. História de usuário

Como bibliotecária chefe (Ana)  
quero ver o relatório de empréstimos  
Para saber quais livros saem mais  

2. Cenário

Dado que escolho o relatório de empréstimos  
Quando o sistema gera o relatório  
Então mostra título do livro, autor e quantidade de empréstimos no período  

Dado que escolho o relatório de empréstimos  
Quando não existem empréstimos no período selecionado  
Então o sistema deve informar que não há registros de empréstimos  

---

3. História de usuário

Como bibliotecária chefe (Ana)  
quero ver relatórios de multas  
Para ver quem está devendo  

3. Cenário

Dado que escolho relatório de multas  
Quando o sistema gera o relatório  
Então mostra nome do usuário, CPF e valor total de multas em aberto  

Dado que escolho relatório de multas  
Quando não existem multas em aberto  
Então o sistema deve informar que não há pendências registradas  

---

4. História de usuário

Como bibliotecária  
quero ver histórico  
Para conferir o que já aconteceu  

4. Cenário

Dado que entro no histórico de empréstimos  
Quando o sistema gera o relatório  
Então aparecem nome do usuário, título do livro, data de empréstimo, data de devolução e status  

Dado que entro no histórico de empréstimos  
Quando não existem registros para o período ou usuário  
Então o sistema deve informar que não há histórico disponível  

---

5. História de usuário

Como bibliotecária  
quero filtrar relatórios  
Para achar informação mais rápido  

5. Cenário

Dado que aplico filtro por período informando data inicial e final ou filtro por usuário informando CPF  
Quando gero o relatório  
Então aparecem apenas registros dentro do período ou vinculados ao usuário informado  

Dado que aplico filtro por período ou usuário  
Quando o filtro não retorna resultados  
Então o sistema deve informar que não há dados para os critérios informados  

---

6. História de usuário

Como bibliotecária  
quero exportar relatório  
Para usar fora do sistema  

6. Cenário

Dado que o relatório de empréstimos, multas ou histórico está na tela  
Quando exporto em PDF ou Excel  
Então o arquivo deve conter exatamente os mesmos campos exibidos no relatório gerado  

Dado que o relatório não foi gerado  
Quando tento exportar  
Então o sistema não deve permitir a exportação  

Dado que o relatório está na tela  
Quando ocorre erro na exportação  
Então o sistema deve informar que não foi possível gerar o arquivo
